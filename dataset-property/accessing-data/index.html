Accessing Data Attributes

Questions to Consider

  • What are the advantages of using data-* attributes in HTML?
    • Keep extra info close to the element it belongs to without extra JS objects.
    • Simple to read/write in JS via element.dataset.
    • Clean separation: you can style with classes and store data with data-*.
    • Works with static HTML and server-rendered pages.

  • How can data-* attributes enhance the accessibility of web applications?
    • Doesn’t add visible text, so you can enhance behavior without cluttering the UI.
    • Lets you map data to ARIA updates (e.g., use data to update aria-label or aria-live).
    • Helps keep the DOM semantic (you don’t need extra wrapper elements just to store data).

  • In what scenarios would you prefer using data-* attributes over other methods of storing data in HTML?
    • Small, element-specific metadata (IDs, categories, flags).
    • When you need data in HTML for progressive enhancement.
    • When a backend template can inject values directly into markup.

  • How can you ensure that the data stored in data-* attributes is valid and secure?
    • Treat data-* as untrusted input; validate in JS before using.
    • Only store non-sensitive info (no secrets or personal data).
    • Sanitize before inserting into the DOM (avoid innerHTML with raw data).
    • Use strict formats (e.g., JSON.parse with try/catch, number checks).
