Canceling a Fetch Request

How to Cancel a Fetch Request

Sometimes, you may want to cancel a fetch request before it completes, for example, if the user navigates away from the page or if the request is taking too long.

This can be achieved using the AbortController API, which allows you to abort one or more DOM requests as and when desired.

Here, we are telling the server to fetch a list of photos. This is a simple fetch request that uses the GET method by default.

Fetching a list of photos from the server can sometimes take time, and you might want to cancel the request if it's no longer needed.

In our eventListener for the "Start Fetch" button, we create an AbortController and pass its signal to the fetch request. If the "Cancel Fetch" button is clicked, we call the abort method on the controller to cancel the request. This will cause the fetch promise to reject with an AbortError. It only works if the fetch request is still pending. If clicked while the fetch is complete, it has no effect. If clicked before the fetch starts, it also has no effect.

  • Use AbortController to cancel a fetch request before it completes.
  • Create an AbortController and pass its signal property to the fetch options.
  • Call controller.abort() to cancel the request. The fetch promise will reject with an AbortError.
