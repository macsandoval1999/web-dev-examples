Fetch API Examples and Notes

What are method, headers, and body?

  • method: The HTTP method (GET, POST, PUT, PATCH, DELETE) tells the server what action to perform.
  • headers: Extra information sent with the request, like Content-Type or authentication tokens.
  • body: The data sent to the server (usually as a JSON string), used with POST, PUT, PATCH.

Fetch API Examples

1. Simple GET Request

In this example, we fetch a single post from the JSONPlaceholder API using a GET request.

We are essentially retrieving data without sending any additional information in the request body.

We are telling the server: "Please send me the post with ID 1."

  • The method is implicitly GET (default for fetch).
  • No headers are needed for this simple request.
  • No body is sent since we are just retrieving data.

2. POST Request with JSON Body

In this example, we create a new post by sending a POST request to the JSONPlaceholder API.

We are sending data to the server to create a new resource.

We are telling the server: "Please create a new post with this title and body."

  • The method is POST, indicating we want to create something new.
  • headers include Content-Type: application/json to inform the server about the data format.
  • body contains the JSON string with the new post's details.

3. PUT Request to Update Data

In this example, we update an existing post by sending a PUT request to the JSONPlaceholder API.

We are sending data to the server to replace the existing resource.

We are telling the server: "Please update the post with ID 1 to have this new title and body."

  • The method is PUT, indicating we want to replace the existing resource.
  • headers include Content-Type: application/json to inform the server about the data format.
  • body contains the JSON string with the updated post's details.

4. PATCH Request to Partially Update Data

In this example, we partially update an existing post by sending a PATCH request to the JSONPlaceholder API.

We are sending data to the server to modify part of the existing resource.

We are telling the server: "Please update the title of the post with ID 1."

  • The method is PATCH, indicating we want to partially update the existing resource.
  • headers include Content-Type: application/json to inform the server about the data format.
  • body contains the JSON string with the fields to be updated.

5. DELETE Request

In this example, we delete an existing post by sending a DELETE request to the JSONPlaceholder API.

We are instructing the server to remove a resource.

We are telling the server: "Please delete the post with ID 1."

  • The method is DELETE, indicating we want to remove the resource.
  • No headers are needed for this simple request.
  • No body is sent since we are just instructing the server to delete the resource.

6. GET Request with API Key

Some APIs require an API key for authentication. This key is usually sent in the request headers.

  • The method is GET, but you can use any method as required by the API.
  • headers include Authorization: Bearer YOUR_API_KEY_HERE (or sometimes x-api-key or another custom header, depending on the API).
  • No body is sent for a GET request.

Note: Replace YOUR_API_KEY_HERE with your actual API key and https://api.example.com/data with the real API endpoint.
